---
name: Phase-based session workflow
description: User works on one implementation phase per session, commits, and restarts for the next phase
type: feedback
---

The user organizes feature work using a plan document split into phases, and works on one phase per session. At the end of each session: update plan and documentation, commit changes, then restart a new session for the next phase.

**Why:** This keeps each session focused and under the context compression threshold, ensuring high-quality output. Commits serve as durable checkpoints between sessions.

**How to apply:** When the user asks to work on a feature, expect a phase-based plan document. Focus on completing one phase per session. At session end, the user will ask to update the plan and commit — do not assume the phase is complete without explicit confirmation.
